# 護理品管資訊系統 (NQIS) 開發需求建議書

## 1. 系統核心設計：考試邏輯模組化 (Exam-Based Logic)
本系統將護理品管行為類比為教育考核體系，確保邏輯清晰、權責分明。

| 角色 | 系統對應項目 | 核心功能說明 |
| :--- | :--- | :--- |
| **教科書** | **照護標準 (SOP)** | 實證基礎、技術規範、臨床路徑的數位化管理。 |
| **小抄** | **衛教教材/臨床小抄** | SOP 的口語轉化版。需與母文件連動，確保衛教一致性。 |
| **考試題目** | **品管監測指標** | 包含指標定義、計算公式、分值權重與合格閾值。 |
| **段考排程** | **監測計畫 (Monitoring)** | 設定稽核頻率、觸發日期、自動隨機抽樣邏輯。 |
| **考生/考場** | **被稽核對象 (Target)** | 系統自動指派的稽核單位、人員或病患個案。 |

---

## 2. 組織架構與動態權限管理
系統採「組別 (Group)」制，以因應大型醫院複雜的組織結構。

* **組織分組：** 病房組、加護組、腫瘤組等。
* **關鍵職位定義：**
    * **總負責人 (Total Lead)：** 具備最終核定權，負責年度計畫與標準定案。
    * **執行幹事 (Executive Officer)：** 負責文件編修、計畫排程與初審流程。
    * **稽核員/審核種子 (Auditor)：** 接收系統派發之「准考證」，執行現場稽核。
    * **單位主管 (HN)：** 負責接收異常通報並提交改善報告 (PDCA)。

## 3. 核心功能模組

### A. 萬用審核與生命週期引擎 (Unified Approval Engine)
* **單一審核表設計：** 所有文件（SOP、考卷、小抄）共用一套簽核邏輯，降低學習成本。
* **生命週期管理：** 涵蓋「制定、修訂、廢止」流程。
* **定期檢審機制：** 系統自動追蹤文件效期（如：兩年一審），逾期自動通知更新。
* **動態關卡路由：** 依據文件「組別」標籤，自動路由至對應的幹事與負責人。

### B. 自動觸發與任務系統 (Trigger & Tasking)
* **自動准考證：** 系統於排程日凌晨，自動根據抽樣邏輯生成「稽核任務」。
* **即時通知：** 透過系統推播或 Email 告知稽核員「稽核地點」與「稽核件數」。

### C. 改善追蹤與 PDCA (Improvement Loop)
* **紅字預警：** 當稽核分數低於合格閾值時，系統自動啟動「改善流程」。
* **分析工具：** 內建 5 Why、魚骨圖分析工具，供單位主管提交檢討報告。
* **審核閉環：** 改善報告需經執行幹事、總負責人審核通過方可結案。

### D. 階層式統計報表 (Advanced Analytics)
* **鑽取式設計 (Drill-down)：**
    * **全院層級：** 關鍵品質指標 (KPI) 達標率趨勢圖。
    * **組別層級：** 組內單位排行與共同指標分析。
    * **單位層級：** 個別指標失分明細與改善歷程。
* **匯出功能：** 支援一鍵生成 PDF 品管月報表與 Excel 原始數據。

## 4. 系統開發優先順序建議

### Phase 1：基礎行政與審核 (MVP)
* 建立組織架構管理。
* 開發「萬用審核表」與「教科書/指標/小抄」的 CRUD 與簽核流程。

### Phase 2：稽核執行自動化
* 開發監測計畫排程器 (Scheduler)。
* 建構稽核任務發送與「行動化稽核」介面。

### Phase 3：改善與分析閉環
* 開發 PDCA 提交與審核模組。
* 開發階層式統計報表與儀表板 (Dashboard)。

### Phase 4：外部系統介接
* 介接 HIS (病歷/給藥資料) 以達成自動數據收集。
* 介接 HR 人事資料以更新角色權限。
